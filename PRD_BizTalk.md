# PRD (제품 요구사항 명세서): 업무 말투 변환기 (BizTalk)

## 1. 프로젝트 개요
- **프로젝트 명**: BizTalk (업무 말투 변환기)
- **목적**: 사용자가 입력한 일상적인 업무 내용을 수신 대상(상사, 동료, 고객 등)에 적합한 비즈니스 전문 용어와 어투로 변환하여 커뮤니케이션 효율을 높임
- **핵심 가치**: "표현 고민에 드는 시간 낭비 방지", "실행 가능한 최소 기능 제품(MVP) 구현"

## 2. 사용자 타겟 및 페르소나
- **신입 사원**: 상사나 타 부서에 메신저/메일을 보낼 때 격식을 차리는 것이 어려운 사용자
- **실무 담당자**: 바쁜 업무 중 정중한 거절이나 요청을 빠르게 문장으로 만들고 싶은 사용자
- **비즈니스 커뮤니케이션 초보**: 핵심 내용은 명확하나 문장 다듬기에 어려움을 느끼는 사용자

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 핵심 기능: 말투 변환 (Tone Transformation)
- **[F-01] 원문 텍스트 입력**: 사용자가 변환하고자 하는 핵심 내용을 자유롭게 입력할 수 있는 텍스트 영역 제공
- **[F-02] 수신 대상 선택**: 변환될 말투의 기준이 되는 수신 대상을 4가지 카테고리 중 하나로 선택 가능
    - **상사/임원**: 격식 있고 공손한 경어체 (예: ~바랍니다, ~여쭤봐도 되겠습니까?)
    - **타팀 동료**: 정중하고 협조적인 업무 어조 (예: ~부탁드립니다, ~확인 부탁드려요)
    - **고객/외부**: 친절하고 신뢰감을 주는 서비스 어조 (예: ~대단히 죄송합니다, ~도와드리겠습니다)
    - **팀 내 동료**: 간결하고 실무 중심적인 어조 (예: ~공유드립니다, ~확인했습니다)
- **[F-03] AI 변환 엔진 연동**: 입력된 텍스트와 선택된 대상을 기반으로 AI(Solar-Pro2)가 적절한 문장으로 변환 수행
- **[F-04] 결과 출력 및 복사**: 변환된 문장을 화면에 표시하고, 클릭 한 번으로 클립보드에 복사할 수 있는 기능 제공

### 3.2 제외 범위 (Out of Scope)
- 사용자 인증 및 로그인 기능
- 변환 기록 저장 및 DB 연동
- 디자인 고도화 및 커스텀 테마

## 4. 기술 스택 (Tech Stack)
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend**: Python, FastAPI
- **AI Engine**: Upstage Solar-Pro2 (LLM)
- **Deployment**: Vercel (프론트엔드 및 전체 배포)

## 5. 시스템 아키텍처 및 데이터 흐름

### 5.1 데이터 흐름
1. **User**: 입력창에 텍스트 입력 및 대상 선택 후 [변환하기] 클릭
2. **Frontend**: API 요청(JSON) 전송 -> `POST /api/convert`
3. **Backend (FastAPI)**: 요청 수신 및 수신 대상별 시스템 프롬프트 구성
4. **AI (Solar-Pro2)**: 프롬프트를 바탕으로 비즈니스 문장 생성
5. **Backend**: 생성된 텍스트를 JSON 형태로 반환
6. **Frontend**: 화면에 결과를 출력하고 사용자가 [복사하기] 클릭 시 클립보드 저장

## 6. UI/UX 요구사항
- **심플한 단일 페이지**: 복잡한 이동 없이 한 화면에서 [입력-선택-결과]가 모두 이루어져야 함
- **직관적인 버튼**: 수신 대상 선택 시 현재 선택된 상태를 시각적으로 명확히 표시
- **반응형 대응**: PC와 모바일 브라우저에서 모두 깨지지 않는 레이아웃

## 7. 단계별 개발 로드맵 (One-Day Project)

### 1단계: 환경 설정 및 백엔드 (90분)
- FastAPI 프로젝트 구조 생성
- Upstage API 키 연동 및 테스트
- 말투 변환용 프롬프트 엔지니어링 (4개 모드)
- `POST /api/convert` 엔드포인트 구현

### 2단계: 프론트엔드 구현 (60분)
- HTML/CSS를 이용한 UI 레이아웃 구성
- Fetch API를 활용한 백엔드 연동 자바스크립트 구현
- 복사하기 및 로딩 상태 처리 기능 추가

### 3단계: 배포 및 검증 (30분)
- GitHub 저장소 푸시
- Vercel을 이용한 라이브 배포
- 실제 업무 케이스(거절, 요청, 보고)별 결과물 테스트

## 8. 완료 기준 (Definition of Done)
- [ ] 텍스트 입력 후 버튼 클릭 시 5초 이내에 변환 결과가 출력되는가?
- [ ] 4가지 대상(상사, 동료, 고객, 팀)별로 결과물의 어투가 명확히 차이나는가?
- [ ] 변환된 결과물이 클립보드에 정상적으로 복사되는가?
- [ ] 배포된 URL을 통해 외부에서도 접속 및 기능 작동이 가능한가?
